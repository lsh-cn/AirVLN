# “天-地”协同智能管护系统（Web端）总体设计方案

> 面向无人机监管与行业应用场景，提供可落地、可扩展、可运维的企业级平台设计。

## 1. 建设目标与设计原则

### 1.1 建设目标
- 打造“飞行管理 + 空域监管 + AI分析 + 风险预警 + 全域可视化”一体化平台。
- 支撑巡检、生态监测、城市治理、应急处置等多场景快速复制。
- 建立从任务规划、执行、分析到闭环处置的全流程数字化能力。

### 1.2 设计原则
- **稳定优先**：关键链路高可用，核心服务支持故障隔离与降级。
- **标准先行**：统一数据标准、接口标准、设备接入标准。
- **云边协同**：边缘实时控制，云端统筹调度与智能分析。
- **安全合规**：飞行安全、数据安全、权限安全、审计可追溯。
- **可扩展**：插件化算法、设备适配器化接入、微服务弹性扩容。

## 2. 总体架构（天-地-云-端）

## 2.1 分层架构
1. **端侧（无人机/机器人/传感器）**  
   机载飞控、RTK、视频回传、载荷（可见光/红外/多光谱）。
2. **边缘层（地面站/边缘网关）**  
   协议适配、低时延控制、视频转码、缓存与断点续传。
3. **平台层（云中心）**  
   任务编排、调度控制、AI分析、空域管理、告警引擎、数据治理。
4. **应用层（Web端大屏/业务工作台）**  
   指挥调度、任务看板、风险预警、专题分析、运维管理。

## 2.2 推荐技术架构（主流、成熟）
- **前端**：Vue3 + TypeScript + Vite + Pinia + ECharts + Mapbox/高德JS SDK。
- **后端**：Java Spring Boot/Spring Cloud（或 Go + Gin）微服务体系。
- **实时通信**：MQTT（设备上行）+ WebSocket（前端实时）+ gRPC（内部高性能RPC）。
- **数据层**：
  - PostgreSQL + PostGIS（业务与空间数据）
  - Redis（缓存/会话/任务状态）
  - Kafka（事件流）
  - MinIO/S3（影像、视频、模型文件）
  - ClickHouse（时序与分析报表）
- **AI基础设施**：PyTorch/ONNX Runtime + Triton Inference Server。
- **运维与交付**：Docker + Kubernetes + Helm + GitLab CI/Jenkins + Prometheus + Grafana + Loki。

## 2.3 核心服务清单
- IAM与租户服务、设备接入服务、任务管理服务、飞控调度服务、空域服务、AI分析服务、告警服务、报表服务、可视化服务、审计与日志服务。

## 3. 核心功能模块设计

## 3.1 飞行任务管理与分析
### 功能
- 任务模板管理（巡检路线、时间窗、载荷策略、返航策略）。
- 任务编排（单机/多机、串并行、条件触发）。
- 任务执行监控（进度、轨迹、载荷状态、电量、链路质量）。
- 数据归档与统计（飞行时长、覆盖率、异常率、工单闭环率）。

### Web端页面
- 任务列表页、任务详情页、任务编排页（可视化流程图）、任务复盘页（轨迹+视频+事件时间线）。

## 3.2 飞行控制与调度管理
### 功能
- 实时遥测：姿态、高度、速度、航向、卫星数、信号质量。
- 远程控制：起降、返航、悬停、航点下发、禁飞区动态规避。
- 调度策略：按区域、机型、电量、优先级自动派单。
- 多机协同：编队巡检、接力飞行、冲突检测与避让建议。

### 安全机制
- 双链路冗余（4G/5G+专网）、控制指令签名、关键动作二次确认。

## 3.3 AI智能监测分析
### 功能
- 多场景识别：烟火识别、违建识别、垃圾堆放、病虫害、河道漂浮物。
- 实时+离线双模：边缘端轻量推理 + 云端高精复核。
- 模型管理：版本管理、A/B测试、在线热更新、效果评估（Precision/Recall/F1）。
- 结果闭环：识别结果自动生成事件并联动工单。

### 数据资产沉淀
- 样本库、标注库、模型库、规则库统一管理。

## 3.4 空域数据管理
### 功能
- 空域图层：禁飞区、限飞区、临时管制区、航线走廊、起降点。
- 数据融合：气象、地形、人口热力、关键设施分布。
- 飞前校验：航线合规检查（空域冲突、气象风险、障碍物风险）。
- 审批协同：飞行申请、审批流、历史留痕与审计。

## 3.5 风险预警与提示机制
### 风险类型
- 飞行风险：低电量、失联、越界、GNSS异常、姿态异常。
- 环境风险：强风降雨、低能见度、雷电、温湿度异常。
- 安全风险：黑飞疑似、敏感区域接近、异常聚集。

### 机制设计
- 规则引擎 + AI异常检测双引擎。
- 告警分级（提示/一般/严重/紧急）与联动预案。
- 多通道通知（Web弹窗、短信、企业微信、语音外呼）。
- 闭环处置（告警-派单-处理-复盘）。

## 3.6 全域资源可视化平台
### 大屏能力
- 全域态势图（设备在线、任务分布、风险热力、空域状态）。
- 指挥大屏（实时视频墙、任务进度、告警中心、资源调度）。
- 经营分析（设备利用率、任务达成率、AI识别准确率、ROI指标）。
- 多层级驾驶舱（省/市/区/网格）逐级下钻。

## 4. 关键业务流程（端到端闭环）

## 4.1 标准巡检流程
1. 制定任务模板与航线。
2. 飞前合规校验（空域+气象+设备健康）。
3. 一键派发任务并自动调度。
4. 飞行中实时监控与异常干预。
5. AI识别生成事件与工单。
6. 处置反馈回流并形成复盘报告。

## 4.2 应急处置流程
- 事件触发 -> 就近资源匹配 -> 快速起飞 -> 视频回传与AI辅助判断 -> 指挥联动 -> 处置归档。

## 5. 数据与接口设计（摘要）

## 5.1 关键数据对象
- `uav_device`：设备信息、机型、能力、状态、维保记录。
- `flight_task`：任务元数据、调度策略、执行状态。
- `flight_telemetry`：时序遥测数据（轨迹、姿态、链路）。
- `ai_event`：识别事件、置信度、位置、截图、视频片段。
- `airspace_zone`：空域要素、多边形范围、时效、管控级别。
- `risk_alert`：预警类型、级别、触发规则、处理状态。

## 5.2 开放API
- RESTful + WebSocket + MQTT Topic规范。
- 支持第三方监管平台、应急平台、视频平台对接。
- 提供SDK（Java/JS/Python）降低项目集成成本。

---

## 6. 稳定性与扩展性保障

## 6.1 稳定性设计
- 微服务分层隔离（控制面/数据面/分析面）。
- 熔断限流、重试补偿、幂等保障、消息削峰填谷。
- 主从与多副本部署、跨可用区容灾、RPO/RTO可配置。
- 全链路可观测（日志、指标、追踪）与自动告警。

## 6.2 扩展性设计
- 设备接入适配器机制（大疆、开源飞控、行业定制机型）。
- AI算法插件化（按行业场景快速装配）。
- 地图图层与业务组件模块化（可按项目启停）。
- 多租户与多组织架构，支持SaaS与私有化双模式。

## 8. 项目落地实施路径（建议）

## 阶段一（0-3个月）：MVP
- 打通设备接入、任务管理、实时监控、基础告警、大屏展示。

## 阶段二（3-6个月）：规模化
- 增加空域管理、审批流、AI识别闭环、报表体系。

## 阶段三（6-12个月）：平台化
- 多租户、算法商城、行业模板、生态开放API。

## 9. 典型行业模板（可快速复制）
- 林业/生态：火情监测、病虫害识别、非法采伐巡查。
- 城市治理：违建巡检、河湖巡查、渣土车与扬尘监管。
- 能源巡检：光伏板缺陷识别、输电通道隐患识别。
- 应急救援：灾害评估、人员搜救、物资投送协同。

---

## 10. 方案价值总结
- **管理价值**：形成“看得见、管得住、可追溯”的全流程监管能力。
- **业务价值**：提升任务执行效率与风险响应速度，降低人工成本。
- **平台价值**：主流技术架构、稳定高可用、模块化扩展，支持快速复制落地。
